# 跨平台路径处理实现总结

## 概述
本项目已完成全面的跨平台路径处理实现，确保在 Windows、macOS 和 Linux 系统上的兼容性。

## 前端实现 (TypeScript/React)

### 1. 路径工具库 (`src/lib/path.ts`)
- **平台检测**: 使用 `navigator.userAgent` 替代已弃用的 `navigator.platform`
- **路径分隔符处理**: Windows 使用 `\`，Unix-like 系统使用 `/`
- **路径标准化**: 统一不同平台的路径分隔符
- **路径连接**: 跨平台安全的路径拼接
- **绝对路径检测**: 支持 Windows 驱动器路径和 UNC 路径
- **文件名清理**: 移除非法字符，处理 Windows 保留名称
- **路径验证**: 检查路径格式合法性
- **环境变量解析**: 处理平台特定的环境变量路径

### 2. 默认路径配置
- **Windows**: `%USERPROFILE%\Documents\CheersAI`
- **macOS**: `~/Documents/CheersAI`
- **Linux**: `~/Documents/CheersAI`

### 3. 应用数据目录
- **Windows**: `%APPDATA%\CheersAI`
- **macOS**: `~/Library/Application Support/CheersAI`
- **Linux**: `~/.config/CheersAI`

## 后端实现 (Rust)

### 1. 文件解析器 (`src-tauri/src/core/file_parser.rs`)
- **智能路径解析**: 支持绝对路径和相对路径
- **多编码支持**: UTF-8、GBK 等编码自动检测
- **错误处理**: 详细的路径查找和错误报告
- **跨平台文件访问**: 标准化的文件读取流程

### 2. 批处理系统 (`src-tauri/src/core/batch.rs`)
- **安全文件名生成**: 跨平台文件名清理
- **Windows 保留名称处理**: 避免系统保留名称冲突
- **路径安全性**: 防止路径注入攻击

### 3. 沙箱管理 (`src-tauri/src/core/sandbox.rs`)
- **跨平台应用数据目录**: 根据操作系统选择合适的数据存储位置
- **文件名清理**: 平台特定的非法字符处理
- **目录创建**: 安全的跨平台目录创建

### 4. 沙箱命令 (`src-tauri/src/commands/sandbox.rs`)
- **目录文件列表**: 支持任意目录的文件枚举
- **安全的 PIN 存储**: 使用 `LazyLock<Mutex<>>` 替代不安全的静态变量
- **时间戳格式化**: 用户友好的时间显示

## 部署注意事项

### 1. Windows 部署
- 确保路径长度不超过 260 字符限制（除非启用长路径支持）
- 处理 Windows Defender 可能的文件访问限制
- 考虑 UAC 权限问题

### 2. macOS 部署
- 处理沙箱权限限制
- 考虑 Gatekeeper 签名要求
- 文件系统大小写敏感性

### 3. Linux 部署
- 不同发行版的路径差异
- 权限和所有权处理
- 文件系统类型兼容性
